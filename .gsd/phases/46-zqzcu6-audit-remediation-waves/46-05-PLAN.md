# S05: LocalCache correctness

**Goal:** Fix issue #3 P1 #10 by making `LocalCache` size/accounting and eviction lifecycle deterministic under concurrency.
**Demo:** Race-enabled cache tests prove bounded size/accounting and clean shutdown under concurrent access.

## Must-Haves

- `LocalCache` has a single lock-owned source of truth for entries and size; overwrites/deletes/expiry cannot drift a separate counter.
- `LocalCache.Close()` stops the background eviction loop and is idempotent.
- Concurrent overwrite/delete tests pass, including under Go race detector for cache package.
- Existing tiered cache behavior remains compatible.
- Main process closes LocalCache during shutdown/error cleanup.
- Go test, cache race test, lint, govulncheck, and artifact UAT pass.
