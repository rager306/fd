# fd ONNX workflow input alignment — M034 S02

## Status

M034 aligns the manual ONNX packaging workflow input contract with M033 provisioning behavior.

TEI remains production/default. ONNX remains explicit opt-in and experimental.

## Workflow change

`.github/workflows/onnx-packaging.yml` no longer hard-fails when `onnx_runtime_source_url` is supplied without `onnx_runtime_sha256`.

Instead:

- `onnx_runtime_sha256` is documented as an optional override;
- when omitted, the workflow logs that provisioning will use `source_contract.onnx_runtime.library_sha256`;
- the provisioning step passes `--onnx-runtime-sha256` only when a non-empty override is supplied.

This matches `tools/provision_onnx_artifacts.py`, which now reads ONNX Runtime library sha from `docs/onnx-artifacts/user-bge-m3-dense-fp32.json` when no CLI override is provided.
