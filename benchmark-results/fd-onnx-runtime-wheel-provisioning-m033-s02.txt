## Status

M033 makes the selected ONNX Runtime wheel source candidate actionable in local/CI provisioning tooling.

TEI remains production/default. ONNX remains explicit opt-in and experimental.

## Change

`tools/provision_onnx_artifacts.py` now reads `source_contract.onnx_runtime` from `docs/onnx-artifacts/user-bge-m3-dense-fp32.json`:

- `library_member`
- `library_size_bytes`
- `library_sha256`

When `--onnx-runtime-source` points to `.whl` or `.zip`, the helper opens the archive, extracts only the configured member, bounds the copy by expected size, writes the runtime destination, and verifies sha256.

For non-zip runtime sources, direct shared-library copy remains supported.
